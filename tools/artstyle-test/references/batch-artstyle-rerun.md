# 批量画风重跑工作流

当画风LoRA格式错误（如缺少`lora:`前缀）导致所有画风描述失效时，需要批量重跑。

## 触发条件

- 发现LoRA格式错误导致画风描述不准确
- 更换了基础模型（如从SDXL切换到NoobAI）
- 公子要求重新评估画风描述

## 脚本位置

`tools/artstyle-test/scripts/artstyle_rerun.py`

## 执行流程

```bash
# 1. 运行重跑脚本（每批15个，约2分钟/批）
python scripts/artstyle_rerun.py --start 0 --limit 15

# 2. 对生成的图片进行vision分析（3路并行，每路5张）
delegate_task → vision分析5张图 → 返回JSON数组

# 3. 将分析结果写入pretags.json
execute_code → 读取pretags.json → 更新画风描述 → 写回

# 4. 统计进度
✅ 更新 15 条画风描述 (累计 X/298)
```

## 批次大小建议

| 批次大小 | 生图时间 | 风险 |
|---------|---------|------|
| 10个 | ~90秒 | 最稳定 |
| 15个 | ~120秒 | 推荐 |
| 20个 | ~180秒 | 可能超时 |

## 并行Vision分析

每批15张图分成3组，每组5张用delegate_task并行分析：
- 任务1: 图片1-5
- 任务2: 图片6-10
- 任务3: 图片11-15

每个任务返回JSON数组格式：
```json
[{"name": "画风名", "desc": "画风描述"}]
```

## 写入格式

画风描述应包含4个维度：
1. **线条风格** — 粗细、流畅度、边缘处理
2. **上色风格** — 平涂/厚涂/渐变、色彩饱和度
3. **光影特征** — 光源方向、明暗对比、阴影类型
4. **整体质感** — 通透/朦胧/锐利、风格定位

## 总耗时估算

298个条目 ÷ 15个/批 = 20批
每批：生图(120s) + vision分析(60s) + 写入(1s) = ~3分钟
总计：~60分钟

## 跳过规则

- 模型文件不存在 → 跳过并记录
- 生图失败 → 跳过并记录
- Civitai模型已下架 → 跳过，后续手动处理
