# Reference 文档标准模板

本模板定义 `references/` 目录下所有参考文档的统一格式。

---

## 📐 文档结构

```markdown
# 文档标题

一句话说明文档目的和适用场景（不超过 50 字）。

## 核心概念

用 3-5 个要点说明核心概念：

- **概念 1** - 简短解释
- **概念 2** - 简短解释
- **概念 3** - 简短解释

## 使用场景

### 场景 1: 场景描述

**步骤**:
1. 步骤 1 - 简短说明
2. 步骤 2 - 简短说明
3. 步骤 3 - 简短说明

**示例** (可选):
\`\`\`bash
# 简短示例
command example
\`\`\`

### 场景 2: 场景描述

（重复上述结构）

## 注意事项

仅列出 2-3 条**最关键**的注意事项：

- ⚠️ 注意事项 1
- ⚠️ 注意事项 2

更多警告见：[WARNINGS.md](../WARNINGS.md)

## 相关文档

- [相关 SKILL](../modules/xxx/SKILL.md)
- [相关 reference](./yyy.md)

---

**最后更新**: YYYY-MM-DD
```

---

## ✅ 编写原则

### 1. 简洁性
- **目标长度**: ≤ 150 行
- **每个章节**: ≤ 30 行
- **步骤说明**: 3-5 步，每步 1 行

### 2. 实用性
- 只保留**核心流程**和**关键决策点**
- 不包含详细代码实现（应在代码文件中）
- 不包含完整示例（SKILL 中已有）

### 3. 时效性
- 不包含历史记录和版本信息
- 不包含"合并自"等元信息
- 不包含时间戳和修正记录

### 4. 准确性
- 所有信息必须准确、可验证
- 过时的信息立即删除
- 不确定的信息不写

---

## ❌ 禁止内容

- ❌ 详细的代码实现（超过 10 行）
- ❌ 完整的 API 文档（应链接到 SKILL）
- ❌ 历史问题和修正记录
- ❌ 个人化语言和临时记录
- ❌ 大量的警告堆叠（应放 WARNINGS.md）
- ❌ 重复 SKILL 中已有的内容

---

## 📋 检查清单

创建/更新 reference 文档前，确认：

- [ ] 长度 ≤ 150 行
- [ ] 遵循标准结构
- [ ] 只包含核心信息
- [ ] 没有重复 SKILL 的内容
- [ ] 没有详细代码实现
- [ ] 注意事项 ≤ 3 条
- [ ] 相关文档链接正确
- [ ] 内容准确、有效、简洁

## 📊 对比示例

### ❌ 不好的例子

```markdown
# 某功能详细指南

本文档合并自以下 5 个文档...

## 历史背景

2026-05-18 公子修正了...

## 详细实现

\`\`\`python
# 100 行的完整代码实现
def some_function():
    # 详细注释...
    ...
\`\`\`

## 注意事项

⚠️ 注意 1...
⚠️ 注意 2...
...（20 条警告）
```

**问题**：
- 包含历史记录
- 代码实现过于详细
- 警告过多

### ✅ 好的例子

```markdown
# 某功能使用指南

通过 X 实现 Y 功能的核心流程和关键决策点。

## 核心概念

- **概念 A** - 简短解释
- **概念 B** - 简短解释

## 使用场景

### 场景 1: 标准用法

**步骤**:
1. 配置环境变量
2. 运行命令
3. 验证结果

### 场景 2: 高级用法

**步骤**:
1. 修改配置
2. 应用自定义参数

## 注意事项

- ⚠️ 必须配置 X 环境变量
- ⚠️ 不支持 Y 场景

更多警告见：[WARNINGS.md](../WARNINGS.md)

## 相关文档

- [功能 SKILL](../modules/xxx/SKILL.md)

---

**最后更新**: 2026-06-07
```

**优点**：
- 结构清晰
- 内容简洁
- 只包含核心信息
